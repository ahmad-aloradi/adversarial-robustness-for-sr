### Pattern 1: Checkpoint Averaging Utility Function
**What:** Standalone function that averages multiple Lightning checkpoint state_dicts
**When to use:** When `use_avg_ckpt=True` in eval.py and no ckpt_path provided
**Example:**
```python
# Source: Adapted from src/callbacks/checkpoint_averaging.py lines 99-136
def average_checkpoints(
    checkpoint_paths: list[Path],
    output_path: Path,
) -> str:
    """Average multiple checkpoint state_dicts and save to output path.

    Args:
        checkpoint_paths: List of paths to .ckpt files to average
        output_path: Path to save the averaged checkpoint

    Returns:
        str: Path to the saved averaged checkpoint
    """
    avg_state: dict[str, torch.Tensor] = {}
    float_keys: list[str] = []
    num = len(checkpoint_paths)

    # Load first checkpoint to initialize
    first_ckpt = torch.load(checkpoint_paths[0], map_location="cpu")
    state_dict = first_ckpt.get("state_dict", {})
    for k, v in state_dict.items():
        if torch.is_tensor(v) and torch.is_floating_point(v):
            avg_state[k] = v.clone().float()
            float_keys.append(k)
        else:
            avg_state[k] = v.clone() if torch.is_tensor(v) else v

    # Accumulate remaining checkpoints
    for ckpt_path in checkpoint_paths[1:]:
        ckpt = torch.load(ckpt_path, map_location="cpu")
        state_dict = ckpt.get("state_dict", {})
        for k in float_keys:
            if k in state_dict and torch.is_tensor(state_dict[k]):
                avg_state[k] += state_dict[k].float()

    # Average
    for k in float_keys:
        avg_state[k] = avg_state[k] / float(num)

    # Save averaged checkpoint (reuse first checkpoint structure)
    first_ckpt["state_dict"] = avg_state
    torch.save(first_ckpt, output_path)

    return str(output_path)
```

### Anti-Patterns to Avoid
- **Using WeightAveraging callback for eval-time averaging:** It's designed for training-time EMA/SWA, not post-hoc averaging of saved checkpoints
- **Averaging non-float tensors:** Only average floating-point parameters; preserve integer buffers (like running counts) as-is from first checkpoint
- **Modifying original checkpoint files:** Always save averaged checkpoint to a new file (e.g., `averaged_topN.ckpt`)

## Don't Hand-Roll

Problems that look simple but have existing solutions:

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Config loading from exp dir | Custom YAML parser | `OmegaConf.load()` | Handles interpolation, type safety |
| Config merging | Dict updates | `OmegaConf.update()` with `merge=True` | Preserves struct, handles nested keys |
| Checkpoint loading | Custom file reader | `torch.load(..., map_location="cpu")` | Handles GPU tensor mapping |
| Path expansion | String manipulation | `Path.expanduser().resolve()` | Cross-platform, handles ~ |

## Common Pitfalls

### Pitfall 1: Averaging Non-Float Tensors
**What goes wrong:** Averaging integer buffers (like batch norm running_mean counts) produces incorrect values
**Why it happens:** Not all state_dict values should be averaged
**How to avoid:** Only average `torch.is_floating_point(v)` tensors; copy others from first checkpoint
**Warning signs:** Unexpected validation/test metrics after loading averaged checkpoint

### Pitfall 2: GPU Memory Spike on Checkpoint Load
**What goes wrong:** Loading checkpoints without `map_location="cpu"` maps GPU tensors to GPU, causing memory spike
**Why it happens:** Default torch.load behavior loads GPU tensors to GPU
**How to avoid:** Always use `torch.load(..., map_location="cpu")` for checkpoint averaging
**Warning signs:** CUDA OOM errors when averaging checkpoints

### Pitfall 3: Experiment Override Injection Timing
**What goes wrong:** Overrides injected after Hydra composition are ignored
**Why it happens:** `@hydra.main` decorator composes config at decoration time, before `main()` runs
**How to avoid:** Call `_inject_exp_dir_overrides()` at module import time (line 325), before `@hydra.main`
**Warning signs:** Missing interpolation errors, config doesn't match training config
**Note:** Already correctly implemented in current code

### Pitfall 4: Hardcoded Checkpoint Filename Patterns
**What goes wrong:** Code assumes `epoch=*.ckpt` pattern but actual names vary
**Why it happens:** ModelCheckpoint filename is configurable
**How to avoid:** Filter by `.ckpt` extension and exclude known files like `last.ckpt`, `averaged_*.ckpt`
**Warning signs:** "Not enough checkpoints" errors when checkpoints exist

### Pitfall 5: Variable Not Defined Before Use
**What goes wrong:** Current code on line 226 references `averaged` but line 223-224 is a TODO
**Why it happens:** Incomplete implementation left in broken state
**How to avoid:** Implement the TODO or raise NotImplementedError
**Warning signs:** `NameError: name 'averaged' is not defined` at runtime
