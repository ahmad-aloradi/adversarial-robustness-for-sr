### Technical Details

**Checkpoint Averaging Algorithm:**
```python
# 1. Load first checkpoint on CPU
first_ckpt = torch.load(checkpoint_paths[0], map_location="cpu")

# 2. Separate float tensors (to average) from others (to copy)
for k, v in state_dict.items():
    if torch.is_tensor(v) and torch.is_floating_point(v):
        avg_state[k] = v.clone().float()  # Accumulator
        float_keys.append(k)
    else:
        avg_state[k] = v.clone() if torch.is_tensor(v) else v

# 3. Accumulate remaining checkpoints
for ckpt_path in checkpoint_paths[1:]:
    ckpt = torch.load(ckpt_path, map_location="cpu")
    for k in float_keys:
        avg_state[k] += state_dict[k].float()

# 4. Divide to get average
for k in float_keys:
    avg_state[k] = avg_state[k] / float(num_checkpoints)

# 5. Save back into first checkpoint structure
first_ckpt["state_dict"] = avg_state
torch.save(first_ckpt, output_path)
```

**Three-Part Auto-Config Flow:**

1. **Override Injection** (`_inject_exp_dir_overrides()`)
   - Called at module import time (line 336)
   - Loads `.hydra/overrides.yaml` from experiment directory
   - Injects overrides into `sys.argv` before Hydra composition
   - Prevents missing/interpolation errors

2. **Config Loading** (`_apply_exp_dir_config()`)
   - Called by `@hydra.main` decorated function
   - Loads `.hydra/config.yaml` from experiment directory
   - Preserves eval-specific settings (predict, use_avg_ckpt, etc.)
   - Merges artifacts and augmentation paths if present

3. **Checkpoint Averaging** (`average_checkpoints()`)
   - Called when `use_avg_ckpt=True`
   - Selects top N checkpoints from experiment directory
   - Creates averaged checkpoint file
   - Uses averaged checkpoint for prediction/testing
