Create `tests/test_bregman_lambda_verification.py` with focused unit tests for the LambdaScheduler. These tests exercise the scheduler directly (no BregmanPruner needed, no trainer mocking needed).

**Test 1: `test_lambda_update_frequency`**
- Create LambdaScheduler with `target_sparsity=0.9, initial_lambda=1e-3`
- Call `step()` 300 times with gradually increasing sparsity (0.5 to 0.9)
- Collect lambda values in a list
- Assert: `len(lambda_values) == 300` (one update per call)
- Assert: all values are finite and positive

**Test 2: `test_lambda_increases_below_target`**
- Create LambdaScheduler with `target_sparsity=0.9, use_ema=False` (disable EMA for deterministic test)
- Call `step(0.5)` multiple times (sparsity well below target)
- Assert: lambda increases monotonically (each value >= previous)
- Assert: lambda > initial_lambda after several steps

**Test 3: `test_lambda_decreases_above_target`**
- Create LambdaScheduler with `target_sparsity=0.5, use_ema=False`
- Call `step(0.8)` multiple times (sparsity well above target)
- Assert: lambda decreases monotonically (each value <= previous)
- Assert: lambda < initial_lambda after several steps

**Test 4: `test_lambda_stable_at_target`**
- Create LambdaScheduler with `target_sparsity=0.9, use_ema=False`
- Call `step(0.9)` (exactly at target)
- Assert: lambda does not change (sparsity_difference == 0, neither branch executes)

**Test 5: `test_lambda_respects_bounds`**
- Create LambdaScheduler with `min_lambda=1e-4, max_lambda=10.0, initial_lambda=1e-3, acceleration_factor=5.0` (aggressive factor to force hitting bounds quickly)
- Call `step(0.1)` many times (far below target, lambda should grow fast)
- Assert: lambda never exceeds `max_lambda`
- Then create another scheduler and call `step(0.99)` many times (far above target)
- Assert: lambda never goes below `min_lambda`

**Test 6: `test_ema_smoothing_reduces_volatility`**
- Create two schedulers: one with `use_ema=True, ema_decay_factor=0.9`, one with `use_ema=False`
- Feed both the SAME alternating sparsity sequence: [0.5, 0.8, 0.5, 0.8, 0.5, 0.8, ...]  (20 steps)
- Record lambda traces from both
- Compute variance of lambda trace for each
- Assert: EMA scheduler has lower variance (smoother) than non-EMA scheduler

**Test 7: `test_checkpoint_save_restore`**
- Create LambdaScheduler, run 50 steps
- Save state via `get_state()`
- Create a NEW LambdaScheduler with DIFFERENT initial params
- Load state via `load_state(saved_state)`
- Assert: `get_lambda()` matches the saved value
- Assert: `get_ema_smoothed_sparsity()` matches the saved value
- Run one more `step()` on both original and restored scheduler with same sparsity
- Assert: both produce the same lambda value (proving state was fully restored)

**Test 8: `test_resume_with_last_sparsity`**
- Create LambdaScheduler with `use_ema=True`
- Run 10 steps, save the EMA value
- Create a new scheduler, call `step(current_sparsity=0.6, last_sparsity=saved_ema)`
- Assert: the EMA is initialized to `last_sparsity`, not to `current_sparsity`
- This tests the resume-from-checkpoint path in `_step_lambda_scheduler`

**Test 9: `test_validation_rejects_invalid_sparsity`**
- Assert that `step(0.0)` raises ValueError (sparsity must be > 0 in Bregman)
- Assert that `step(-0.1)` raises ValueError
- Assert that `step(1.5)` raises ValueError
- Assert that `step(float('nan'))` raises ValueError
- Assert that `step(float('inf'))` raises ValueError

Run all tests with: `pytest tests/test_bregman_lambda_verification.py -v`

```python
def _make_bregman_pruner_and_mocks(target_sparsity=0.9, initial_lambda=1e-3):
    """Create a BregmanPruner with lambda scheduler and mock trainer."""
    scheduler = LambdaScheduler(
        initial_lambda=initial_lambda,
        target_sparsity=target_sparsity,
        use_ema=False,
    )

        sparsity_threshold=1e-30,
        verbose=0,
        lambda_scheduler=scheduler,
    )
    return pruner, scheduler
```
